Design System

Position

position property specifies the type of positioning method used for an element

There are five different position values:

  • Static
  • Relative
  • Fixed
  • Absolute
  • Sticky

Element are then positioned using the top, bottom, left and right properties. However, these properties wil not work unless the position property is set first.

Static

is not positioned in any special way; it is always positioned according to the normal flow of the page

This div element has position static. It's cant be affect by position elements

.static-example {
position: static;
border: 3px solid black
}

Relative

an element with relative position, is positioned to its normal position

setting the top, right, bottom, left properties of a relatively-positioned element will cause it to be ajusted away from its normal position.

This div element has position relative.

.relative-example {
position: relative;
left: 30px
border: 3px solid black
}

fixed

is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. the top, right, bottom, and left properties are used to position the element.

This div element has position: fixed

.fixed-example {
position: fixed;
bottom: 0;
right: 0;
width: 300px;
border: 3px solid black
}

Absolute

absolute positioned element are removed from the normal flow, and can overlap elements

this div element has position relative

This div element has position absolute

.relative-absolute {
position: relative;
width: 400px;
height: 200px;
border: 3px solid black
}
.aboslute-example {
position: absolute;
top: 80px;
right: 0;
width: 200px;
height: 100px;
border: 3px solid black
}

Sticky

is positioned based on the user's scroll position

This div element has position: sticky

.sticky-example {
position: sticky;
background-color: green top: 0;
border: 2px solid black
}

Display

The display property is the most important CSS property for controlling layout

Every HTML element has a default display value depending on what type of element it is. The default display value for most elements is block or inline.

Block element

A block level element always starts on a new line and takes up the full width available

The div element is a block level element.

Examples of block elements:

  • div
  • h1 - h6
  • form
  • p
  • header
  • footer
  • section

Inline elements

An inline element does not start on a new line and only takes up as much width as necessary.

This is an inline span element inside a paragraph

Examples of inline elements:

  • span
  • a
  • img

Inline-block

compare to inline, the major difference is that inline-block allows to set a width and heigh on the element.

also with inline-block the top and bottom margins/paddings are respected.

Compare to block, the major difference is that inline-block does not add a line-break after the element, so the element can sit next to other elements

None

none is commonly used with javascript to hide and show elements without deleting and recreating them.
